Componentes en Angular:

Definición:

Un componente en Angular es una unidad lógica y reutilizable de una interfaz de usuario. Un componente encapsula la lógica y la presentación de una parte específica de la aplicación.

Características Principales:

  • Plantilla (Template): Los componentes tienen una plantilla asociada que define la estructura HTML de la interfaz de usuario.
  • Estilos: Pueden tener estilos CSS específicos asociados.
  • Lógica: Contienen la lógica de la aplicación en el archivo TypeScript asociado.
  • Metadatos (Metadata): Los metadatos del componente, como decoradores, proporcionan información adicional sobre el componente.

Directivas Estructurales en Angular:

Definición:

Las directivas estructurales son instrucciones que alteran la estructura del DOM (Document Object Model). Permiten controlar la visibilidad y la existencia de elementos en la interfaz de usuario.

Directivas Estructurales Comunes:

  • ngIf: Se utiliza para mostrar o ocultar un elemento basado en una expresión condicional.
  • ngFor: Se utiliza para repetir elementos en función de una expresión iterable.
  • ngSwitch: Se utiliza para mostrar elementos diferentes basados en expresiones.

Novedades y Características de Angular CLI 13.0.0

Angular Cache

La CLI ahora guarda su caché en disco por defecto, lo que mejora el rendimiento al ejecutar comandos como ng serve o ng build. Para habilitarlo, ejecutar el siguiente comando en la terminal: 

ng config cli.cache.enabled true

Para habilitarlo en todo el proyecto modificando el archivo angular.json: 


            {{codigoTres}}

Esta característica reemplaza el antiguo NG_PERSISTENT_BUILD_CACHE.

Diferential Loading Eliminado

La carga diferencial, introducida en Angular CLI 8.0, se ha eliminado en la versión 13.0.0. Dado que IE11 ya no es compatible, ng build ya no realiza la carga diferencial y solo emite código ES2015+.

Automático Migraciones

Algunos esquemas de Angular actualizarán automáticamente los archivos de configuración al actualizar a la versión 13.0.0.

Eliminación de Internet Explorer 11

Dado que IE11 ya no es compatible, se ha eliminado la configuración relacionada con la compatibilidad con Internet Explorer en la CLI.

Eliminación de Comandos de Producción

Los comandos de producción, como ng build-prod, han sido eliminados. Ahora se puede utilizar un único comando, ng build, para construir la aplicación en modo de producción.
